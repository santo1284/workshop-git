control de versiones

Un control de versiones es un sistema que registra cada cambio que se realiza en el código fuente de un proyecto. Esto te permite tener un histórico de todos los cambios producidos en sus ficheros, saber quién lo hizo y cuándo.

que es GIT

Git es un sistema de control de versiones distribuido que permite a los desarrolladores gestionar y rastrear los cambios en el código fuente a lo largo del tiempo. Fue creado por Linus Torvalds en 2005 para mejorar el desarrollo del kernel de Linux.

¿Qué es un repositorio?, repositorio locales y remotos.

un repositorio es un contenedor para el código y su historial de cambios. Los repositorios locales permiten el desarrollo individual y cómputo offline, mientras que los repositorios remotos facilitan la colaboración y el trabajo en equipo en proyectos de software.

repositorio local

Un repositorio local es una copia del repositorio de un proyecto que reside en la máquina del desarrollador. A diferencia de un repositorio remoto, que se encuentra en un servidor o en una plataforma de colaboración (como GitHub o GitLab), el repositorio local permite al desarrollador trabajar de manera independiente, sin necesidad de conexión a Internet.

repositorio remoto

Un repositorio remoto es una versión de un repositorio de proyecto que se aloja en un servidor o en una plataforma de desarrollo colaborativa, como GitHub, GitLab, Bitbucket o Azure DevOps. Este tipo de repositorio permite que varios desarrolladores colaboren en el mismo proyecto al compartir y sincronizar sus cambios.

que es GitHub

GitHub es una plataforma de desarrollo de software que utiliza Git como sistema de control de versiones. Se ha convertido en una de las herramientas más populares para el desarrollo colaborativo de proyectos de software debido a sus múltiples características y funcionalidades.

que es una rama GIT y para qeu sirve

Una rama en Git es una línea de desarrollo independiente dentro de un repositorio. Las ramas permiten a los desarrolladores trabajar en diferentes características, correcciones de errores u otras tareas sin afectar la rama principal (generalmente llamada main o master).

sirve para: Una rama en Git sirve para facilitar el desarrollo de software al permitir que los desarrolladores trabajen en nuevas funciones o correcciones de errores de manera aislada, sin afectar el código estable en la rama principal. Esto es especialmente útil para experimentar con nuevas ideas o implementar cambios grandes sin el riesgo de romper el código existente.

cuales son los estados en git

  1. Modificado: Cuando realizas cambios en un archivo en tu directorio de trabajo, ese archivo pasa al estado "modificado". Esto significa que has editado el archivo, pero aún no has preparado esos cambios para la confirmación.
  2. Prepara (staged): Cuando decides que quieres incluir los cambios de un archivo modificado en tu próxima confirmación, utilizas el comando git add. Esto cambia el estado del archivo a "preparado". En este estado, Git está listo para incluir los cambios en el próximo "commit".
  3. Confirmado (committed): Después de preparar los cambios, los confirmas utilizando el comando git commit. Esto guarda los cambios en el repositorio y los marca como parte del historial de versiones. Los archivos en este estado están almacenados en el repositorio y son parte de la versión actual del proyecto.

¿Qué es la terminal, consola o línea de comandos y para qué sirve?

La terminal, consola o línea de comandos son interfaces que permiten a los usuarios interactuar con el sistema operativo de una computadora mediante texto, en lugar de utilizar una interfaz gráfica. Estas herramientas son comúnmente utilizadas en sistemas operativos como Linux, macOS y Windows (a través de la "Símbolo del sistema" o "PowerShell").
